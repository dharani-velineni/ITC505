Lab 3: Serving a Custom Font

The Process of Serving a Custom Font

To serve a custom font to a browser, you can use the @font-face rule in CSS. This allows you to specify a custom font that is hosted on your server and deliver it to the client's browser when they visit your webpage. Here’s how you can do it:

  1. Choose a font: First, pick a font that isn’t likely to be on most users’ systems. You can download free fonts from various online resources like google fonts.
  2. Convert the font to web formats: Ensure the font is available in web-friendly formats like .woff, >.tff and .woff2. You can use an online tool like Font Squirrel’s Webfont Generator to handle the conversion (if needed).
  3. Upload the font: Upload the font files to a folder on your web server, such as /fonts.
  4. Add CSS: Use the @font-face rule to define your font in your CSS. The src property links to the font files on your server, and you can specify fallback fonts in case the custom font fails to load.
  5. Apply the font: Once the font is defined, use the font-family property to apply it to specific elements or your whole page.

For example, here’s the CSS I used to serve the custom font "CrazyFont" on this page:

        @font-face {
            font-family: 'CrazyFont';
            src: url('fonts/Creepster-Regular.ttf') format('truetype');
            font-weight: normal;
            font-style: normal;
        }

        body {
            font-family: 'CrazyFont', sans-serif;
        }

See the Custom Font in Action

If everything is working correctly, the text on this page should look pretty unusual, demonstrating that the custom font is being successfully served from the server.
